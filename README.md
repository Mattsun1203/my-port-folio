# プロジェクト構成

## アーキテクチャ
モノレポ構成でjavascriptと相性の良いpnpmを使用。npmはファントム依存が発生したり、バージョン管理が発生しややこしいため。
AI駆動開発をする場合、モノレポならアプリケーションを跨いだ実装も整合性を取りながらやってくれるので、メリットが大きいと思っている。

## パッケージ

###　アプリケーション共通

#### Biome
linter&format用パッケージ
eslint&prettier構成も悩んだが、バージョン管理がややこしくなるためBiomeにした。
ただ今までの実績からeslint&prettierの方が動作は安定するので、biome一択ではないと思っている。

#### lefthook
preCommit,prePush用パッケージ。
特に比較は行っていないが、あまり困ったことがなかったので採用

### フロントエンド(ポートフォリオサイト)

#### next.js
ISRで構築。(マイクロCMSと連携してコンテンツ投稿できるようにしたいのでSSGはNG。SSRがベストだと思うが、ISRを試用)

#### tailwind-css
好みはあるが、個人的には一つのファイルでまとまってくれた方がわかりやすい。
慣れるとUIを作るスピードが速い。
と言う点でかなり良いと思っている。

#### MSW
モックライブラリ。
アプリケーション側のコードに影響を及ぼないと言う点で、これ一択だと思っている。

#### tanstack-query
キャッシュ用ライブラリ。
慣れれば可読性にも良い。

### バックエンド

#### nest.js
jsで統一したかったのでこれ一択。
linterやフォーマットの設定が共有できるのが大きい。
